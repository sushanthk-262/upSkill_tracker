Transformers & Attention

Week 1

The Attention Mechanism — Intuition

Attention lets every token "look at" every other token when being encoded. It's what makes transformers understand context.

Q = query  (what am I looking for?)
K = key    (what do I contain?)
V = value  (what do I actually contribute?)

Attention(Q,K,V) = softmax(QK^T / √d_k) × V

# √d_k = scaling to prevent vanishing gradients
# softmax → weights sum to 1 (attention distribution)
# "The cat sat on the mat" — when encoding "sat",
# attention weight on "cat" is high (who sat?)

Multi-head: Run h attention heads in parallel, each learning different relationships (syntax, coreference, semantics). Concatenate results.

Transformer Architecture

Input Text
    ↓ Tokenizer
Token IDs [1, 9453, 281, ...]
    ↓ Embedding layer
Token Embeddings (d_model dimensions)
    ↓ + Positional Encoding
    ↓ N × Encoder/Decoder Blocks:
       - Multi-Head Self-Attention
       - Add & Norm (residual)
       - Feed Forward (2 linear layers + ReLU)
       - Add & Norm
    ↓ Output layer (vocab size logits)
    ↓ Softmax
Next Token Probabilities
  • Encoder-only: BERT — classification, embeddings
  • Decoder-only: GPT, Llama, Claude — text generation
  • Encoder-Decoder: T5, BART — translation, summarization

Context Window & Generation

  • Context window: Max tokens the model can "see" at once (input + output)
  • Temperature: 0 = deterministic, 1 = creative, >1 = chaotic
  • Top-p (nucleus): Sample from top-p probability mass (0.9 = diverse)
  • Top-k: Sample from top-k most likely tokens
  • Max tokens: Hard cap on output length
  • Stop sequences: Strings that halt generation early
Interview tip: Know that KV-cache stores computed attention keys/values to avoid re-computing on each new token. This is why longer contexts are expensive.

Embeddings

Weeks 4–5

What Are Embeddings?

Dense vector representations of text. Semantically similar text → vectors that are close in high-dimensional space.

from openai import OpenAI
client = OpenAI()

def embed(text: str) -> list[float]:
    response = client.embeddings.create(
        model="text-embedding-3-small",  # 1536 dims
        input=text
    )
    return response.data[0].embedding

# Cosine similarity = how similar two vectors are
import numpy as np
def cosine_sim(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

v1 = embed("Python programming")
v2 = embed("coding in Python")
v3 = embed("recipe for pasta")
print(cosine_sim(v1, v2))  # ~0.95 (very similar)
print(cosine_sim(v1, v3))  # ~0.3  (dissimilar)

RAG Pipeline

Weeks 5–8

RAG Architecture

Ingestion (offline):
  Documents → Chunking → Embedding → Vector Store

Query (online):
  User question
      ↓ Embed query
      ↓ Similarity search in vector store
  Top-k relevant chunks
      ↓ Combine: chunks + question → prompt
      ↓ LLM generates answer
  Response with citations

# Basic RAG from scratch
def rag_query(question: str, vectorstore, llm):
    # 1. Retrieve
    docs = vectorstore.similarity_search(question, k=4)
    context = "\n\n".join([d.page_content for d in docs])

    # 2. Augment
    prompt = f"""Answer based ONLY on this context:
    {context}

    Question: {question}
    Answer:"""

    # 3. Generate
    return llm.invoke(prompt)

HyDE — Hypothetical Document Embeddings

def hyde_retrieval(question: str, vectorstore, llm):
    # Step 1: Generate a hypothetical answer
    hypothetical = llm.invoke(
        f"Write a short passage that would answer: {question}"
    )

    # Step 2: Embed the hypothetical answer (not the question!)
    embedding = embed(hypothetical)

    # Step 3: Search with that embedding
    docs = vectorstore.similarity_search_by_vector(embedding, k=4)
    return docs

# Why it works: hypothetical answer is in the same
# semantic space as real documents, even if factually wrong
# → better retrieval for abstract queries

Reranking with Cohere

import cohere
co = cohere.Client(api_key=os.environ["COHERE_API_KEY"])

def rerank(query: str, documents: list[str], top_n=3):
    response = co.rerank(
        model="rerank-english-v3.0",
        query=query,
        documents=documents,
        top_n=top_n,
    )
    return [documents[r.index] for r in response.results]

# Pipeline: retrieve k=20, rerank to top 3
# Much cheaper than embedding 20 docs every time
# Cross-encoder (reranker) > bi-encoder (embedding)
# for final scoring — but too slow for first-stage

ANN Search Algorithms

  • HNSW (Hierarchical NSW): Graph-based. Fast, accurate. Default in most vector DBs. O(log n) query.
  • IVF (Inverted File): Cluster-based. Good for huge datasets (>1M vectors). Used by Faiss.
  • Flat (brute force): Exact search. Perfect accuracy, O(n). Only viable for small datasets.
  • Product Quantization: Compress vectors to reduce memory. Trades accuracy for speed.

For interviews: know that vector DBs use Approximate Nearest Neighbor search — trading tiny accuracy loss for massive speed gains at scale.

    issues = llm.invoke(f"Find issues in:\n{state['code']}")

def generate_fix(state: AgentState) -> dict:
    fix = llm.invoke(f"Fix these issues:\n{state['issues']}")
    return {"code": fix}

    if state["iteration"] >= 3 or not state["issues"]:

    return "fix"

### Longest Substring Without Repeats — O(n)
def lengthOfLongestSubstring(s):
    seen = {}; left = maxLen = 0
    for right, c in enumerate(s):
        if c in seen and seen[c] >= left:
            left = seen[c] + 1
        seen[c] = right
        maxLen = max(maxLen, right - left + 1)
    return maxLen

### Two Sum (sorted) — O(n)
def twoSum(numbers, target):
    l, r = 0, len(numbers) - 1
    while l < r:
        s = numbers[l] + numbers[r]
        if s == target: return [l+1, r+1]
        elif s < target: l += 1
        else: r -= 1

BFS + Topological Sort

from collections import deque, defaultdict

### Course Schedule (cycle detection)
def canFinish(n, prerequisites):
    adj = defaultdict(list)
    indegree = [0] * n
    for a, b in prerequisites:
        adj[b].append(a); indegree[a] += 1
    q = deque(i for i in range(n) if indegree[i]==0)
    done = 0
    while q:
        node = q.popleft(); done += 1
        for nb in adj[node]:
            indegree[nb] -= 1
            if indegree[nb] == 0: q.append(nb)
    return done == n

DP Patterns

### Coin Change — O(amount × coins)
def coinChange(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for coin in coins:
        for x in range(coin, amount + 1):
            dp[x] = min(dp[x], dp[x-coin] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1

### LIS — O(n log n)
import bisect
def lengthOfLIS(nums):
    tails = []
    for n in nums:
        pos = bisect.bisect_left(tails, n)
        if pos == len(tails): tails.append(n)
        else: tails[pos] = n
    return len(tails)

Heap + Backtracking

import heapq

### Kth Largest — O(n log k)
def findKthLargest(nums, k):
    heap = nums[:k]
    heapq.heapify(heap)
    for n in nums[k:]:
        if n > heap[0]: heapq.heapreplace(heap, n)
    return heap[0]

### Subsets — O(2^n)
def subsets(nums):
    result = []
    def bt(start, path):
        result.append(path[:])
        for i in range(start, len(nums)):
            path.append(nums[i])
            bt(i+1, path)
            path.pop()
    bt(0, [])
    return result

import fitz  # PyMuPDF
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_openai import OpenAIEmbeddings
from pinecone import Pinecone

async def ingest_pdf(file_bytes: bytes, doc_id: str):
    # 1. Extract text
    pdf = fitz.open(stream=file_bytes, filetype="pdf")
    pages = [page.get_text() for page in pdf]
    full_text = "\n\n".join(pages)

    # 2. Chunk
    splitter = RecursiveCharacterTextSplitter(
        chunk_size=1000, chunk_overlap=200
    )
    chunks = splitter.split_text(full_text)

    # 3. Embed
    embedder = OpenAIEmbeddings(model="text-embedding-3-small")
    embeddings = await embedder.aembed_documents(chunks)

    # 4. Upsert to Pinecone (namespace = doc_id)
    index = Pinecone().Index("rag-docs")
    vectors = [
        {"id": f"{doc_id}-{i}", "values": emb,
         "metadata": {"text": chunk, "doc_id": doc_id, "chunk": i}}
        for i, (chunk, emb) in enumerate(zip(chunks, embeddings))
    ]
    index.upsert(vectors=vectors, namespace=doc_id)
    return {"chunks": len(chunks)}

                          namespace=doc_id, include_metadata=True)

import ast, inspect

def extract_functions(source_code: str) -> list[dict]:
    tree = ast.parse(source_code)
    functions = []
    for node in ast.walk(tree):
        if isinstance(node, (ast.FunctionDef, ast.AsyncFunctionDef)):
            functions.append({
                "name": node.name,
                "args": [a.arg for a in node.args.args],
                "docstring": ast.get_docstring(node) or "",
                "returns": ast.unparse(node.returns) if node.returns else None,
                "source": ast.unparse(node),  # full function source
                "lineno": node.lineno,
                # Extract type hints for better test generation
                "annotations": {
                    a.arg: ast.unparse(node.args.annotations[i])
                    for i, a in enumerate(node.args.args)
                    if i < len(node.args.annotations)
                }
            })
    return functions

def validate_tests(test_code: str, source_code: str) -> dict:

from tree_sitter import Language, Parser
import tree_sitter_python as tspython

PY_LANGUAGE = Language(tspython.language())
parser = Parser(PY_LANGUAGE)

def parse_python_file(source: str) -> list[dict]:
    tree = parser.parse(bytes(source, "utf8"))

    functions = []
    # Query: find all function definitions
    query = PY_LANGUAGE.query("""
        (function_definition
          name: (identifier) @name
          parameters: (parameters) @params
          body: (block) @body)
    """)

    for pattern, captures in query.matches(tree.root_node):
        name_node  = captures.get("name")
        body_node  = captures.get("body")
        if name_node and body_node:
            functions.append({
                "name": source[name_node.start_byte:name_node.end_byte],
                "source": source[body_node.start_byte:body_node.end_byte],
                "start_line": name_node.start_point[0] + 1,
            })
    return functions

LLM API Quick Ref

GPT-4o: $2.50/1M input | 128k ctx | best general
Claude Sonnet: $3.00/1M | 200k ctx | best for code/docs
GPT-4o-mini: $0.15/1M | 128k ctx | production scale
Gemini 1.5 Pro: $1.25/1M | 1M ctx | massive context
temp=0: deterministic | temp=1: creative
top_p=0.9: diverse | top_k=50: constrained

RAG Pipeline Quick Ref

Chunk size: 1000 tokens overlap: 200 (baseline)
Embedding: text-embedding-3-small → 1536 dims
Code embedding: voyage-code-2 (better for capstone)
Retrieve: k=20 → rerank → top 3
Eval: Faithfulness, Answer Relevancy, Context Recall
Hybrid search: BM25 (0.4) + Semantic (0.6) weights

Agent Cheat Sheet

Chain: fixed steps, fast, predictable
Agent: dynamic steps, tool selection, flexible
ReAct: Thought → Action → Observation loop
LangGraph: stateful, conditional routing, persistence
Human-in-loop: interrupt_before=[node] in LangGraph
Max iterations: always set to prevent infinite loops

Fine-tuning Quick Ref

When: consistent format/style; not for knowledge
LoRA rank r: 16 typical | higher = more capacity
QLoRA: 4-bit base + LoRA = 4× less VRAM
7B model: ~5GB VRAM (4-bit) — fits on RTX 3080
Min data: ~500–1000 examples for good fine-tune
Tools: PEFT + Unsloth (fast) + Axolotl (prod)
